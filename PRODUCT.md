# Product

## Register

product

## Users

CS students learning design patterns for the first time and experienced developers referencing patterns during work. Both arrive with intent — either following a structured curriculum or searching for specific guidance. They use laptops or desktops, often in focused sessions, sometimes task-switching during implementation.

## Product Purpose

A visual learning platform for all 23 Gang of Four design patterns. It exists because static textbooks and dense UML diagrams make patterns harder to grasp than they need to be. Success means someone moves from "I've heard of this pattern" to "I know when to use it and can implement it" — and returns later for reference without friction.

## Brand Personality

Precise, calm, authoritative. The voice is direct and knowledgeable — it assumes smart users, never condescends. Like a well-edited technical reference, every word earns its place. The interface recedes so content leads; confidence comes from clarity, not decoration.

## Anti-references

- **Gamified / cartoonish**: No badges, XP, streaks, mascots, confetti, or "level up" language. This is not Duolingo.
- **Enterprise / legacy Java tutorials**: No Oracle-style boxes-and-arrows, no 2000s academic formatting, no beige backgrounds with Comic Sans code blocks.
- **SaaS marketing**: No gradient CTAs, pricing tables, testimonial carousels, "start free trial" urgency. The product is the learning, not a funnel.
- **Terminal-native / hacker aesthetic**: Not a green-on-black monospace wall. The surface has warmth and hierarchy, not raw console.

## Design Principles

1. **Show, then tell.** Interactive visualizations and live diagrams lead; explanatory text supports. A diagram that clarifies instantly beats a paragraph that takes 30 seconds.
2. **Expert confidence.** The interface assumes the user is capable. Explanations are thorough but never remedial. Copy is precise, not padded.
3. **Calm focus.** The UI recedes. Generous whitespace, restrained motion, purposeful color — nothing competes with the content the user came to read.
4. **Progressive depth.** The surface gives a fast answer (what is this pattern, when do I use it). Drilling deeper reveals intent, UML, code, and pitfalls. Every journey starts shallow and goes deep on demand.
5. **Consistency as clarity.** Category colors, icon metaphors, structural templates — once you learn the system on pattern #1, pattern #23 requires no re-learning.

## Accessibility & Inclusion

WCAG 2.1 AAA. All interactive visualizations provide keyboard-accessible alternatives. Reduced motion is respected (animations degrade to crossfade or instant transitions). Color is never the sole differentiator — category distinctions use shape, label, and position as redundant channels. Code blocks and diagrams meet ≥7:1 contrast. Focus management is explicit and predictable across all interactive surfaces.
